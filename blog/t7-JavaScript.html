Classes vs. Constructor FunctionsApril 11, 2015

Similar but not quite the same...

Alrighty, we are going to look at the fundamental differences between Ruby's classes and JavaScript's constructor functions. I am going to keep this very simple and (hopefully) very short.

Ruby classes are pretty easily defined. You start off with:

class Greet

.....def var

........@var

.....end

.....def var=(val)

........@var = val

.....end

end

different_greeting = Greet.new

This will get you a new version of Greet that has the same basic values already programmed into it because you have called #new. If we were to try to do this in JavaScript, it would look something like this:

var greet = {};

greet.greeting = "Hello!"

Now, using this setup, we can add as many properties to our object as we would like to. This is very similar to a Ruby hash. They are both used to store key/value pairs. It is easier to access these pair in Ruby if we are trying to get at them with a loop than it is if we are trying to get into them with a loop in JavaScript, though. In Ruby, we have the .each function that is not present in JavaScript. We would have to actually go create a function in JavaScript and specifically tell it how to get into the object and pull out only what we are looking for. This could be pretty difficult, depending on what kind of calculations we are trying to do.
